Loomforge Architecture

Local workflow engine for agentic software delivery

Workflow Pipeline

  • • Build → Review → optional Revise → Push
  • • One revision pass per issue, then ship
  • • Builder and reviewer configurable per project
  • • Auto-PR from dev → main on project completion

Integration Points

  • • MCP server for OpenClaw agent integration
  • • CLI for operator access
  • • Linear SDK for issue fetch and status sync
  • • GitHub CLI for push and PR creation

Runtime

  • • TypeScript · Node 22+ · Fastify
  • • SQLite for run state and events
  • • File-based artifact storage
  • • launchd-managed local daemon
