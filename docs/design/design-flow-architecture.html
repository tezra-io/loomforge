Loomforge Design Flow Architecture

Requirement → scaffolded repo + reviewed design doc + Linear project

Pipeline Stages

  • • Validate inputs (slug pattern, requirement source)
  • • Scaffold dir + git init + governance files
  • • Optional gh repo create (if gh CLI available)
  • • Draft → review → at most one revise → publish

Inputs & Surfaces

  • • loomforge design new <slug> (CLI)
  • • loomforge design extend <slug> --feature ...
  • • HTTP POST /design/new · /design/extend
  • • MCP loom_design_* tools (OpenClaw)

Outputs & Handoff

  • • Local repo with reviewed design doc + AGENTS/CLAUDE.md
  • • Linear project + Linear Document
  • • Project entry appended to ~/.loomforge/loom.yaml
  • • needs_remote flag if no GitHub remote was created
